This project was developed as a hands-on lab assignment within the **FreeCodeCamp** Responsive Web Design curriculum, focusing on form validation, pseudo-class list selectors, and deep UI component customization.

## 🎯 Project Objective & Requirements (FreeCodeCamp)

The core objective was to build a form that strictly satisfies automated technical specifications while implementing clean, production-ready UI design.

### Fulfilled Technical Criteria (User Stories):
- [x] **Semantic Structure:** Containerized within a layout .container block, featuring logical field groupings.
- [x] **Strict Input Associations:** Every interactive element explicitly binds to an accessible text label.
- [x] **Focus State UX (:focus):** Tailored focus ring behavior across inputs and textareas to substitute default browser outlines with a matching theme border.
- [x] **Dynamic Validation (:valid / :invalid):** Configured strict sequential list selectors (input, select, textarea) to instantly flag valid entries (green) and form errors (red).
- [x] **Overridden Custom Radio UI:** Stripped user-agent default radio button rendering using appearance: none. Re-engineered the active indicator with custom shadows, background animations, and adjacent sibling selectors (+) to highlight selected text labels dynamically.
- [x] **Structural Accents (:first-of-type):** Targeted the initial name input field independently to create an isolated visual design variant.
- [x] **Interactive Submissions:** Added smooth transition animations to the primary submit action using :hover states.

1. Clone the repository using git clone command followed by your repository URL:

2. Navigate into the project folder using your terminal:
   cd job-application-form

3. Open the project:
